# 集中兵力

> "对于人，伤其十指不如断其一指；对于敌，击溃其十个师不如歼灭其一个师。"
> —— 毛泽东《中国革命战争的战略问题》（1936年）

## 核心原则

**资源有限时，不要分散力量同时应对多个问题，而要集中优势兵力于主要矛盾，彻底解决一个再转向下一个。不打无准备之仗，不打无把握之仗。**

## 不适用场景

- 当前只有一个任务，不存在资源竞争
- 用户明确要求同时推进多个任务
- 任务是并行独立的，完全没有资源竞争
- 已经在执行选定的主攻任务，不需要重新决策

## 方法流程

### 第一步：列出所有待解决问题

> "先打分散和孤立之敌，后打集中和强大之敌。"

全面盘点当前面对的所有任务和问题：规模、难度、紧迫程度、依赖关系。

### 第二步：用矛盾分析法确定主要矛盾

找出主要矛盾——那个解决了就能带动其他问题的关键问题。

### 第三步：选择突破口

1. **先易后难** —— 先解决有把握的问题，积累信心和资源
2. **先孤后强** —— 先解决孤立的、不被其他问题支撑的问题
3. **先关键后外围** —— 如果某个问题是其他问题的根源，优先解决它
4. **有准备才开打** —— 确保对选定的问题有充分了解

### 第四步：集中全部力量，彻底解决

- **不要分心** —— 暂时搁置其他问题
- **追求彻底** —— "伤其十指不如断其一指"
- **快速解决** —— 在局部问题上追求速决
- **验证完成** —— 确认问题确实被彻底解决了

### 第五步：转向下一个目标

一个问题彻底解决后：短暂总结经验 → 评估剩余问题是否发生变化 → 重新确定下一个目标。

## 十大军事原则映射

| 军事原则 | 通用场景映射 |
|---------|-----------|
| 先打弱敌后打强敌 | 先解决有把握的任务，积累势能 |
| 先取小城后取大城 | 先拿下小成果，再攻坚大目标 |
| 以歼灭有生力量为目标 | 追求彻底解决问题，而非临时绕过 |
| 集中绝对优势兵力 | 同一时间只聚焦一个核心问题 |
| 不打无准备之仗 | 准备充分才行动 |
| 连续作战作风 | 解决完一个马上转向下一个 |
| 战役间隙休整 | 阶段性总结和调整 |

## 操作规程

当本方法被触发时，**输出任务优先级矩阵并宣告主攻方向**：

1. **列出所有待处理任务**，为每个任务标注：
   - 影响范围：高/中/低
   - 解决难度：高/中/低
   - 依赖关系：是否被其他任务所依赖

2. **选择主攻目标**（一次只选一个），优先级标准：
   - 首选：影响范围高 + 被其他任务依赖（解锁阻塞）
   - 次选：影响范围高 + 解决难度中（最高性价比）
   - 末选：解决难度低（积累势能）

3. **宣告并锁定**：
   ```
   🎯 当前主攻目标：[任务名]
   原因：[一句话说明为何优先此任务]
   完成信号：[什么情况下认为完成]
   暂缓任务：
   - [任务B]：[为何暂缓]
   - [任务C]：[为何暂缓]
   ```

4. **执行期间规则**：新需求先判断是否属于当前主攻范围，不属于则记录后暂缓。
