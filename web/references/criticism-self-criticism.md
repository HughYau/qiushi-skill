# 批评与自我批评

> "房子是应该经常打扫的，不打扫就会积满了灰尘；脸是应该经常洗的，不洗也就会灰尘满面。我们同志的思想，我们党的工作，也会沾染灰尘的，也应该打扫和洗涤。"
> —— 毛泽东《论联合政府》（1945年）

## 核心原则

**定期的、诚实的自我审视和相互批评是防止错误积累的唯一机制。批评的目的不是惩罚，而是"治病救人"。**

## 不适用场景

- 任务刚开始，还没有可以审视的工作成果
- 用户在实时指导中，反馈已经即时纳入
- 简单的单步任务
- 已处于紧急修复的执行阶段 —— 先完成修复，结束后再审视

## 方法流程

### 第一步：先自我批评

> "对以前的错误一定要揭发，不讲情面"

在请别人审查之前，先诚实地审视自己的工作：
- 我的工作目标完成了吗？
- 过程中有哪些地方做得不够好？
- 有没有偷懒、敷衍、走捷径的地方？
- 有没有我知道有问题但选择忽略的地方？
- 如果重新来过，我会在哪里做不同的选择？

**关键纪律：不讳疾忌医。**

### 第二步：具体而非笼统

> 批评必须是"以科学的态度来分析批判"

- **错误示范：** "我觉得做得不够好"
- **正确示范：** "在第三步我跳过了验证环节，导致后面出现了 X 问题"

### 第三步：基于事实而非猜测

批评必须建立在调查研究的基础上：引用具体的事实和证据。

### 第四步：治病救人，不是整人

> "完全是为了救人，而不是为了把人整死。"

每指出一个问题，就提出一个改进建议。关注"下次怎么做更好"。

### 第五步：欢迎他人批评

> "不管是什么人，谁向我们指出都行。只要你说得对，我们就改正。"

接收批评时：先完整听取，不急于辩解；判断是否基于事实；正确则接受并改正。

### 第六步：形成常态机制

批评与自我批评不是应急措施，而是日常机制。

## 操作规程

当本方法被触发时，**必须输出结构化的审视报告**：

```
## 工作审视报告

### 原定目标
[复述原本要完成的目标和验收标准]

### 完成情况
- [x] 已完成：……
- [ ] 未完成：……（原因：……）

### 发现的问题
| 严重程度 | 问题描述（具体行为，非笼统描述） | 根本原因 | 改进建议 |
|---------|-------------------------------|---------|---------|
| 必须改正 | … | … | … |
| 应当改正 | … | … | … |
| 建议改进 | … | … | … |

### 做得好的地方
- ……

### 下次重点关注
- ……
```

**填写规则：**
- "问题描述"必须包含具体的步骤编号或文件名，禁止写"做得不够好"这类无信息量的描述
- "根本原因"必须指向一个方法论层面的失误（跳过了调查？没有验证？分散了注意力？）
- 如果"发现的问题"表格为空，必须明确写出"本次工作未发现需要改正的问题，原因是……"

## 附录：工作审查检查清单

### 一、完整性检查
- [ ] 原定目标是否全部达成？
- [ ] 是否有遗漏的需求或子任务？
- [ ] 边界情况和异常路径是否处理？
- [ ] 相关文档/注释是否同步更新？
- [ ] 是否有"先跳过后面再补"但实际未补的内容？

### 二、正确性检查
- [ ] 核心逻辑是否经过验证？
- [ ] 修改是否引入了新问题？
- [ ] 假设和前提条件是否仍然成立？
- [ ] 数据流和状态变更是否正确？

### 三、方法论检查
- [ ] 是否做了充分的调查研究再行动？
- [ ] 是否抓住了主要矛盾？
- [ ] 方案是否经过实践验证？
- [ ] 是否收集了相关方的意见？
- [ ] 资源是否集中在最重要的问题上？

### 四、质量检查
- [ ] 代码/方案是否清晰可读？
- [ ] 是否存在不必要的复杂性？
- [ ] 是否有"差不多就行了"的敷衍之处？
- [ ] 如果重新来过，会在哪里做出不同选择？
