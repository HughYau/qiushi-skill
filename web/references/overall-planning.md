# 统筹兼顾

> "提出这十个问题，都是围绕着一个基本方针，就是要把国内外一切积极因素调动起来，为社会主义事业服务。"
> —— 毛泽东《论十大关系》（1956年）

## 核心原则

**复杂系统中存在多对辩证关系，不能只顾一头而忽略另一头。正确的方法是识别所有重要的关系对，在它们之间找到动态平衡，最大限度地调动一切积极因素。**

## 不适用场景

- 只有一个目标维度，不存在需要平衡的对立面
- 用户已经明确了取舍方向
- 属于战术执行阶段，大方向已定
- 问题的性质是"必须选一个放弃另一个"的硬性约束

## 方法流程

### 第一步：识别所有辩证关系对

常见的辩证关系对：
- 速度 与 质量
- 深度 与 广度
- 短期收益 与 长期价值
- 简单 与 灵活
- 核心功能 与 附加功能
- 探索创新 与 稳定可靠
- 集中统一 与 分散自主

**注意：** 不同任务的辩证关系对不同，要根据具体情况识别。

### 第二步：拒绝片面性

审视自己是否正在片面地强调某一方面。
**关键检验：** 如果你正在做的事完全忽略了某一对关系中的一方，那就是片面性。

### 第三步：寻找动态平衡

平衡不是固定的折中，而是根据具体条件的动态调整：
- 在当前阶段，哪一方需要更多关注？
- 条件变化后，平衡点是否需要调整？

### 第四步：学习他人教训

参考类似场景中他人的失败教训。但借鉴不是照搬——必须结合自己的具体情况调整。

### 第五步：调动一切积极因素

统筹兼顾的最终目标不是"各方都少一点"的妥协，而是找到**让各方面都能发挥积极作用**的配置。

### 第六步：系统性思考

调整一对关系时，检查对其他关系的影响。**每次调整都要检查全局影响。**

## 常见的辩证关系处理模式

| 关系对 | 片面做法 | 统筹兼顾做法 |
|-------|---------|------------|
| 速度 vs 质量 | 只赶进度不管质量 | 关键点严格把关，非关键点适当加速 |
| 深度 vs 广度 | 钻牛角尖或浮于表面 | 先广泛了解，再在关键点深入 |
| 短期 vs 长期 | 只看眼前或好高骛远 | 解决眼前问题时考虑长期影响 |
| 简单 vs 灵活 | 过度简化或过度设计 | 当前够用，预留扩展空间 |
| 探索 vs 稳定 | 不敢创新或过度冒险 | 在稳定基础上有计划地探索 |

## 操作规程

当本方法被触发时，**输出辩证关系全景图和平衡方案**：

1. **识别所有辩证关系对**：
   ```
   当前涉及的辩证关系：
   1. [A] ↔ [B]：目前偏向 [A/B/均衡]，偏向程度：[轻微/明显/严重]
   2. [C] ↔ [D]：目前偏向 [C/D/均衡]，偏向程度：……
   ```

2. **片面性检查**：当前方案是否完全忽略了某一方？刻意取舍还是无意忽略？

3. **确定当前阶段的平衡点**：
   ```
   [A] vs [B]：当前阶段优先 [A]，但确保 [B] 不低于 [最低限度]
   原因：……
   ```

4. **系统影响检查**：
   ```
   如果加强 [A]，对其他关系的影响：
   - 对 [C] vs [D]：[加剧/缓解/无影响]
   ```

5. **设置失衡预警指标**：
   ```
   如果出现 [具体信号]，说明 [A] vs [B] 已经失衡，需要重新调用此方法
   ```
