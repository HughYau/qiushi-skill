# 群众路线

> "将群众的意见（分散的无系统的意见）集中起来（经过研究，化为集中的系统的意见），又到群众中去作宣传解释，化为群众的意见，使群众坚持下去，见之于行动，并在群众行动中考验这些意见是否正确。如此无限循环，一次比一次地更正确、更生动、更丰富。"
> —— 毛泽东《关于领导方法的若干问题》（1943年）

## 核心原则

**正确的认识和决策来自双向的知识流动：从实践者那里收集分散的意见，经过系统化整理后返回给实践者验证和执行，如此循环往复，一次比一次更正确。**

## AI 语境下的"群众"

在 AI agent 的工作语境中，"群众"是**一切持有真实情况的信息源**：

| 原始含义 | AI 语境映射 |
|---------|-----------|
| 人民群众的实际经验 | 代码库中现有的模式、惯例和注释 |
| 基层的第一手信息 | git history、测试用例、错误日志 |
| 群众对方案的反馈 | 运行结果、编译错误、测试失败 |
| 多方意见的综合 | 多个信息源的交叉验证 |

## 不适用场景

- 任务信息来源单一且完整
- 已经完成了充分的多源信息收集，处于执行阶段
- 紧急修复且问题来源明确
- 用户是唯一信息源且其输入已经足够精确

## 方法流程

### 第一步：收集——从群众中来

深入到实践者/使用者中去，收集他们的实际需求、看法、经验和想法。
- 收集到的意见通常是分散的、无系统的、甚至互相矛盾的——这是正常的
- 不要急于评判或筛选，先全面收集
- 要主动去问，不要被动等待反馈

### 第二步：集中——系统化研究

对收集到的分散意见进行：
- 归类整理——找出共性和差异
- 分析研究——用矛盾分析法找出核心问题
- 提炼综合——将分散意见系统化为连贯的方案
- 去粗取精——保留有价值的洞察，去除偏见和误解

**关键：** 系统化不是简单的多数投票。要运用分析能力，将零散智慧提炼为有条理的认识。

### 第三步：返回——到群众中去

将系统化的方案返回给实践者/使用者：
- 用他们能理解的方式解释方案
- 说明方案是如何从他们的意见中提炼出来的
- 征求他们对系统化方案的反馈

### 第四步：检验——在实践中验证

方案在实践中执行后，观察效果。

### 第五步：再收集——开始新一轮循环

根据实践检验的结果，重新收集意见，开始新一轮循环。

## 两种错误倾向

### 命令主义（忽视群众意见）
自己制定方案，不征求使用者意见。违背了"从群众中来"的原则。

### 尾巴主义（只跟随不整理）
用户说什么就做什么，不加分析。违背了"集中起来"的原则。群众路线不是简单地服从多数，而是要把分散意见**系统化、提升**后再返回验证。

## 操作规程

当本方法被触发时，**在给出任何方案之前，先执行多源信息收集**：

1. **识别信息源清单**：
   - 用户的明确需求（用户说了什么）
   - 代码库的现状（实际情况是什么）
   - 运行结果/日志（现实反馈是什么）
   - 相关文档/注释/git history（历史经验是什么）

2. **从各源收集事实**（不是解释，是事实）

3. **系统化整理**：识别各信息源之间的一致之处、矛盾之处、信息缺口

4. **形成并返回判断**：
   ```
   综合各方信息后，我的判断是：……
   这个判断的依据是：[信息源1] + [信息源2] 共同指向……
   与用户原始描述有出入的地方：……（如有）
   ```

5. **验证闭环**：方案实施后，检查结果是否与判断一致
